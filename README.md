## 🔒 Hard Rules (Read Carefully)

### ✅ Rule 1: Sequential Phases Only
- **Cannot skip** Phase 1 → Phase 2
- **Cannot skip** Phase 1 & 2 → Phase 3
- This exists for a reason

### ✅ Rule 2: All Hours Must Be Logged
- No "thinking about code" counts
- Every hour must have specifics (what you did)
- Minimum: 10 hours/week

### ✅ Rule 3: Projects Must Pass Code Review
- All projects on GitHub
- >80% test coverage required
- Senior engineer code review approval

### ✅ Rule 4: No 2-Week Gaps
- Missing a week is OK
- Missing 2 consecutive weeks breaks momentum
- Get back on track immediately

### ✅ Rule 5: Document Everything
- Module summaries required
- ADRs for Phase 2+
- Blog posts in Phase 3
- If it wasn't documented, it didn't happen

## 🚨 When You Get Stuck

### What's Normal
- Weeks 2-3: "What am I doing?" is normal
- Month 2: Boredom with fundamentals is expected
- Month 3: First real win feels amazing
- Month 6: You're building real things

### What Requires Action
- Missing 2 consecutive weeks
- Phase taking 50%+ longer than planned
- Serious blockers (give 2 weeks then ask for help)

### How to Get Help
1. Check the [FAQ](#faq) below
2. Re-read the phase module docs

## 🤔 FAQ

### Q: Can I go faster than 3-4.5 years?
**A**: Not at 10 hours/week. To go faster, invest more time. Quality takes time.

### Q: Can I do multiple phases?
**A**: No. Phase sequencing exists because Phase 2 requires Phase 1 foundation.

### Q: What if I miss a week?
**A**: Catch up next week. Don't let 2 weeks pass.

### Q: Should I review code?
**A**: Yes! Essential for Phase 1 completion. Ask seniors to review your work.

### Q: Is this realistic?
**A**: Yes. This follows real Principal Engineer paths. The timeline matches industry reality.

### Q: What if I want to quit?
**A**: Review your [COMMITMENT](COMMITMENT.md). This path is long - make sure it's truly what you want.
