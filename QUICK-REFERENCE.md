## 🚀 Your Daily Workflow

### Morning (Start of work day)
1. Open [current Phase Module](roadmap/PHASE-1-MODULES.md)
2. Find today's task
3. Start working

### During Day
1. Use [TIME-TRACKING.md](roadmap/TIME-TRACKING.md) to log hours
2. Push code to GitHub
3. Take breaks every 90 minutes

### Evening (End of work day)
1. Update your time log
2. Commit code
3. Write one line in your journal about what you learned

### Weekly (Sunday evening)
1. Read [ROADMAP-PROGRESS.md](ROADMAP-PROGRESS.md)
2. Copy [WEEKLY-CHECK-IN-TEMPLATE.md](journal/WEEKLY-CHECK-IN-TEMPLATE.md)
3. Fill in weekly review
4. Update [PROJECTS-COMPLETED.md](PROJECTS-COMPLETED.md)
5. Plan next week from Phase Modules

### Monthly (Last day of month)
1. Verify all hours in [TIME-TRACKING.md](roadmap/TIME-TRACKING.md)
2. Update main [ROADMAP-PROGRESS.md](ROADMAP-PROGRESS.md)
3. Check if you're on pace (should have ~40 hours done)
4. Adjust next month if needed

## ⚡ The 5 Hard Rules (Never Break These)

1. **Phases are sequential only**  
   Can't skip Phase 1 → complete before Phase 2  
   Can't do phases in parallel  

2. **Hours must be verified**  
   Log daily, be honest  
   No 2-week gaps  
   Minimum 10 hours/week  

3. **Projects are real**  
   Working code on GitHub  
   >80% test coverage  
   Code reviewed  
   No shortcuts  

4. **Skip nothing**  
   Must complete ALL items marked ✅  
   No "I'll come back to it"  

5. **Document everything**  
   Blog posts for each project  
   Weekly reflections  
   GitHub commits with messages

## 💡 Pro Tips

1. **Bookmark this page** in your browser  
   Use Cmd+D (Mac) or Ctrl+D (Windows)

2. **Print Phase Modules** for your current week  
   Read offline, annotate by hand

   - Every Monday: New week planning

4. **Share progress monthly**  
   Write blog post or Twitter thread  
   Keeps you accountable

5. **Join communities early**  
   Week 1, not Week 12  
   You need support
